Release Version Date Modules

Every quarter, Oracle updates an organisation's environments with fixes and new features. Changes may include customer enhancements, legislative changes and updates to the underlying technology stack.

An organisation's cohort determines when the quarterly update is applied to their pods:

RELEASE
Cohort A B C D
A Feb May Aug Nov
B Mar Jun Sep Dec
C Apr Jul Oct Jan

Stage pods are updated on the first Friday of an organisation's quarterly update month. Production pods are updated two weeks later on the third Friday of the month.

Quarterly update names combine the year and A, B, C or D. For example, the release for the first quarter of 2023 is 23A; the release for the second quarter of 2023 is 23B; and the release for the first quarter of 2024 will be 24A.

Quarterly updates cannot be delayed or skipped. This ensures that all customers use the most up-to-date and certified versions of the applications.

Source: Oracle Blog: Harness the Business Value of Quarterly Updates
